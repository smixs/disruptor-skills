# Как допиливать Disruptor

Цель проекта — собрать **ультрамашину-дизраптор агентной разработки на реальной
практике**. Не теория: каждый метод здесь выстрадан на живых проектах. Присылай своё.

## Что особенно ждём

- **Новый метод/стадию** — если у тебя есть выстраданная дисциплина, которой во флоу
  не хватает. Добавляй как референс в `references/` и впиши его в конвейер `SKILL.md`.
- **Правку существующего метода** — из реального прогона, где он сломался или недосказал.
- **Новый on-ramp или гейт** — если есть класс задач, который входит во флоу иначе.
- **Кросс-доменные примеры** — где метод сработал вне веба.

## Правила, чтобы это остался флоу, а не свалка практик

1. **Каждый референс держит шапку `## Stage contract`** (`Inputs → Outputs → Entry gate
   → Done when → Next`). Без неё стадия не вплетается в конвейер.
2. **Outputs стадии N = Inputs стадии N+1.** Проверь стыкуемость.
3. **Method vs prompt.** Метод — дисциплина, которой следуешь сам. Prompt — шаблон,
   который заполняешь и отдаёшь исполнителю. Не смешивай.
4. **Источник — as is.** Если приносишь чужую наработку из чата — сохраняем её целиком,
   с атрибуцией автору.
5. **KISS/YAGNI.** Новая стадия должна зарабатывать своё место. Деталь — это поле в
   таблице, а не новый файл.

## Поток

1. Форкни, ветка `feat/<что>`.
2. Изменения + шапка контракта.
3. PR с одним логическим изменением. Опиши, на каком реальном прогоне это выстрадано.
